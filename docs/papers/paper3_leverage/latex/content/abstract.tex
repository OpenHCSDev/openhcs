This paper derives a decidable optimization criterion for coherent architectures: maximize leverage
\[L(A)=\frac{|\mathrm{Capabilities}(A)|}{\mathrm{DOF}(A)}.\]

Building on axis orthogonality (Paper 1) and Single Source of Truth (Paper 2), we prove a family of results connecting DOF, error probability, and modification cost. Key contributions include:

- DOF–Reliability Isomorphism: an architecture with $n$ DOF is isomorphic to an $n$-component series reliability system, preserving failure-ordering.
- Leverage–Error Tradeoff: among architectures with equal capabilities, larger $L$ implies strictly smaller expected error probability.
- Modification Complexity Gap: at fixed capabilities, expected modification cost scales with DOF (constant at optimal leverage, linear above it).
- Optimality Criterion: given requirements $R$, the architecture that maximizes $L$ subject to feasibility constraints minimizes expected error; the selection is computable and decidable.

We illustrate these results with instances (SSOT vs scattered checks, nominal vs duck typing, service decomposition) and connect the theory to implementation patterns. All core theorems are machine-checked in Lean 4; proofs and an assumption ledger accompany the paper. Practical validation is provided by an OpenHCS migration case study that demonstrates the predicted leverage and localization gains.

Keywords: software architecture, leverage, degrees of freedom, reliability, formal methods
