\section{Introduction}\label{sec:introduction}

Consider any decision problem $\mathcal{D}$ with actions $A$ and factored state space $S = X_1 \times \cdots \times X_n$. A coordinate set $I \subseteq \{1, \ldots, n\}$ is \emph{sufficient} if knowing only coordinates in $I$ determines the optimal action:
\[
s_I = s'_I \implies \Opt(s) = \Opt(s')
\]
This is \emph{coordinate sufficiency}, the formalization of \emph{information sufficiency} (whether available information determines the optimal action) over a factored state space.

In information-theoretic terms, this is a task-conditional coding problem: the object preserved by projection is the decision boundary $s \mapsto \Opt(s)$, not the full source state. The same object appears in classical source-coding and fidelity viewpoints \cite{shannon1948mathematical,shannon1959coding,cover2006elements,slepian1973noiseless}, and in zero-error/confusability structure where exact recoverability depends on graph constraints rather than raw description length \cite{shannon1956zero,korner1973graphs,lovasz1979shannon,csiszar2011information}. The same separation also appears in model selection: short descriptions and certifiable adequacy are distinct \cite{rissanen1978modeling,grunwald2007minimum,li2008introduction,blahut1972computation}.

For physical instantiations, the same tuple has a direct substrate reading: coordinates are measurable state degrees of freedom, actions are physically realizable interventions or transitions, and the optimizer map is the decision boundary induced by the declared objective contract. The physical-to-core encoding discipline in Section~\ref{sec:physical-transport} makes this bridge theorem-level rather than metaphorical.\claimstamp{P\ref{prop:physical-claim-transport},\ref{prop:law-instance-objective-bridge},\ref{prop:time-discrete},\ref{prop:decision-unit-time},\ref{prop:run-time-accounting},\ref{prop:substrate-unit-time};C\ref{cor:physical-counterexample-core-failure};T\ref{thm:physical-bridge-bundle}}{AR}

\begin{remark}[Universality of the Decision-Problem Tuple]\label{rem:universality}
The formal object of this paper is the decision-problem tuple $\mathcal{D} = (A, S, U)$: actions, factored states, utility. Within this model family, any problem requiring a choice among alternatives given structured information instantiates the tuple: the alternatives are $A$, the structured information is $S = X_1 \times \cdots \times X_n$, and the objective is $U$. Thus the qualifier ``decision problem'' does not narrow scope \emph{inside the declared coordinate model}. The meta-problem results below are universal over problems representable with this coordinate structure; the $(A, S, U)$ formalism supplies the language for stating the sufficiency predicate.
\end{remark}

\subsection{Landscape Summary}

\textbf{When is sufficiency checking tractable?} We identify six sufficient conditions:

\begin{enumerate}
\item \textbf{Bounded actions} ($|A| \leq k$) under explicit-state encoding: with constantly many actions, we enumerate action pairs over the explicit utility table.

\item \textbf{Separable utility} ($U(a,s) = f(a) + g(s)$): The optimal action depends only on $f$, making all coordinates irrelevant to the decision.

\item \textbf{Low tensor rank}: When utility has bounded tensor rank $R$, factored computation yields $O(|A| \cdot R \cdot n)$ complexity.

\item \textbf{Tree-structured utility}: With explicit local factors over a tree, dynamic programming yields polynomial algorithms in the input length.

\item \textbf{Bounded treewidth}: When pairwise interactions form a graph of treewidth $w$, CSP algorithms yield $O(n \cdot k^{w+1})$ complexity.

\item \textbf{Coordinate symmetry}: When utility is invariant under coordinate permutations, orbit types reduce the effective state space to $\binom{d+k-1}{k-1}$ equivalence classes, polynomial in $d$ for fixed $k$.
\end{enumerate}

Each condition is stated with its encoding assumption. Outside these regimes, the general problem remains \coNP-hard (Theorem~\ref{thm:sufficiency-conp}).

\textbf{When is it intractable?} The general problem is \coNP-complete (Theorem~\ref{thm:sufficiency-conp}), with a separation between explicit-state tractability and succinct worst-case hardness:
\begin{itemize}
\item In the explicit-state model: SUFFICIENCY-CHECK is solvable in polynomial time in $|S|$ by explicitly computing $\Opt(s)$ for all $s\in S$ and checking all pairs $(s,s')$ with equal $I$-projection. In particular, instances with $k^* = O(\log |S|)$ are tractable in this model.
\item In the intermediate query-access model: the finite-state core yields an Opt-oracle lower bound $\Omega(|S|)$ (Boolean instantiation: $\Omega(2^n)$), and the mechanized Boolean interface refinements preserve the obstruction for value-entry and state-batch interfaces, with explicit subproblem-to-full transfer, weighted randomized robustness, and oracle-lattice transfer/strictness theorems (Propositions~\ref{prop:query-regime-obstruction}--\ref{prop:oracle-lattice-strict}).
\item In the succinct model: under ETH there exist worst-case instances produced by our strengthened gadget in which the minimal sufficient set has size $\Omega(n)$ (indeed $n$) and SUFFICIENCY-CHECK requires $2^{\Omega(n)}$ time.
\end{itemize}

The explicit ETH lower bound is still a succinct worst-case statement; Proposition~\ref{prop:query-regime-obstruction} provides the finite-state query core and Propositions~\ref{prop:query-value-entry-lb}--\ref{prop:oracle-lattice-strict} provide theorem-level Boolean-interface refinements.

\subsection{Main Theorems}

\begin{enumerate}
\item \textbf{Theorem~\ref{thm:sufficiency-conp}:} SUFFICIENCY-CHECK is \coNP-complete via reduction from TAUTOLOGY.
